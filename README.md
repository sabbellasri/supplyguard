## рҹ”Ң Data Sources (Coral Architecture)

### рҹ—„пёҸ Supabase вҖ” Supplier Master + Compliance
```sql
SELECT s.id, s.name, s.country, c.iso9001_expiry, c.audit_score
FROM suppliers s
LEFT JOIN compliance_certs c ON c.supplier_id = s.id
WHERE s.active = true ORDER BY c.audit_score ASC;
```

### рҹҡў AfterShip вҖ” Shipment Tracking
```
GET /v4/trackings?filter[status]=Exception,InfoReceived
вҶ’ Returns: holds, delays, port exceptions per supplier
```

### рҹ“Ҡ Google Sheets вҖ” Manual Audit Log
```
=IMPORTRANGE("supplier-audits-2024","Audits!A2:H")
вҶ’ Filtered: score < 70 OR critical_findings > 0
```

### рҹҢҚ GDACS вҖ” Live Geo Disaster Alerts *(real endpoint)*
```
GET https://www.gdacs.org/xml/rss.xml
вҶ’ Parses Orange/Red alerts, correlates to supplier countries
```

## рҹӨ– Claude AI Integration

SupplyGuard calls `claude-sonnet-4-20250514` with streaming enabled. The prompt includes:

- Supplier name, country, category
- Composite risk score + per-signal breakdown
- Which Coral connectors are currently active
- All signal details from the four sources

Claude returns a structured **3-part risk brief**:
1. **SITUATION** вҖ” core risk in plain language
2. **CRITICAL FACTORS** вҖ” the 2вҖ“3 most urgent issues
3. **RECOMMENDED ACTION** вҖ” a clear directive

> The analysis changes dynamically as you toggle connectors on/off вҖ” Claude always knows which sources are live.
